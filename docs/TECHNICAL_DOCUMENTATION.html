OS Concept: Threads

What is a Thread?

A thread is a unit of execution within a process. Multiple threads share the same memory space but have their own stack and registers.

Why Use Threads?

  • Parallelism: Utilize multiple CPU cores
  • Responsiveness: Keep UI responsive while doing work
  • Efficiency: Share memory instead of IPC

Thread Creation in POSIX

// Creating a thread
pthread_t thread_handle;
pthread_create(&thread_handle, nullptr, thread_function, argument);

// Thread function signature
void* thread_function(void* arg) {
    // Do work...
    return nullptr;
}

// Waiting for thread to finish
pthread_join(thread_handle, nullptr);

OS Concept: Mutexes & Critical Sections

âŒ The Problem: Race Conditions

// DANGEROUS - Race condition!
int counter = 0;

void* increment(void* arg) {
    for (int i = 0; i < 1000000; i++) {
        counter++;  // NOT atomic!
    }
    return nullptr;
}
// Two threads might give counter < 2000000!

âœ… The Solution: Mutexes

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
int counter = 0;

void* safe_increment(void* arg) {
    for (int i = 0; i < 1000000; i++) {
        pthread_mutex_lock(&mutex);    // Acquire lock
        counter++;                      // Critical section
        pthread_mutex_unlock(&mutex);  // Release lock
    }
    return nullptr;
}
// Now counter will always be exactly 2000000

OS Concept: Condition Variables

âŒ The Problem: Busy Waiting

// BAD - Wastes CPU!
while (!runway_available) {
    // Spin, checking repeatedly
}

âœ… The Solution: Condition Variables

pthread_mutex_t mutex;
pthread_cond_t runway_clear;

void wait_for_runway() {
    pthread_mutex_lock(&mutex);
    
    while (!runway_available) {
        // Sleep until signaled
        pthread_cond_wait(&runway_clear, &mutex);
    }
    
    runway_available = false;
    pthread_mutex_unlock(&mutex);
}

void release_runway() {
    pthread_mutex_lock(&mutex);
    runway_available = true;
    pthread_cond_signal(&runway_clear);  // Wake one waiter
    pthread_mutex_unlock(&mutex);
}

OS Concept: Deadlock

ğŸ’€ What is Deadlock?

Deadlock occurs when threads are waiting for each other in a cycle:

Thread A holds Resource 1, waiting for Resource 2
Thread B holds Resource 2, waiting for Resource 1
â†’ Neither can proceed â†’ DEADLOCK!

Four Conditions for Deadlock

  1. Mutual Exclusion: Resources can only be held by one thread
  2. Hold and Wait: Thread holds resources while waiting for more
  3. No Preemption: Resources cannot be forcibly taken
  4. Circular Wait: Cycle of threads waiting for each other

Prevention: Resource Ordering

// Define global order
enum ResourceOrder {

    TUG = 5
};

// All threads acquire in this order - no deadlock possible!

OS Concept: Virtual Memory

ğŸ“ What is Virtual Memory?

Virtual memory gives each process the illusion of having its own large, contiguous memory space. Memory is divided into fixed-size pages mapped to physical frames.

Address Translation

Virtual Address: [Page Number | Offset] | v Page Table Lookup | v Physical Address: [Frame Number | Offset]

OS Concept: TLB (Translation Lookaside Buffer)

A TLB is a hardware cache of recent page table entries:

Virtual Address | v TLB Lookup (fast, ~1 cycle) | HIT? â”€â”€Yesâ”€â”€> Physical Address | No (TLB Miss) | v Page Table Lookup (slow, ~100 cycles) | v Update TLB, return Physical Address 
class TLB {
    TLBEntry entries[64];  // Fixed size
    int hits = 0, misses = 0;
    
    int lookup(int virtual_page) {
        for (int i = 0; i < 64; i++) {
            if (entries[i].valid && 
                entries[i].virtual_page == virtual_page) {
                hits++;
                return entries[i].physical_frame;  // HIT!
            }
        }
        misses++;
        return -1;  // MISS
    }
};

OS Concept: Page Replacement

Clock Algorithm (Second Chance)

When physical memory is full, the clock algorithm finds a victim page to evict:

int ClockReplacer::find_victim() {
    while (true) {
        if (frames[clock_hand].reference_bit == 0) {
            int victim = clock_hand;
            clock_hand = (clock_hand + 1) % frames.size();
            return victim;  // Found victim
        }
        
        // Give second chance
        frames[clock_hand].reference_bit = 0;
        clock_hand = (clock_hand + 1) % frames.size();
    }
}
ğŸ’¡ Why Clock? O(1) amortized time, approximates LRU, simple to implement.
